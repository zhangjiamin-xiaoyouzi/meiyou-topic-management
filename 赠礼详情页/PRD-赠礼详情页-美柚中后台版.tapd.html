一、需求背景

1.1 业务大背景

美柚会员体系正在逐步完善，当前已具备会员开通、权益展示等基础能力。为进一步提升会员价值感和留存率，计划引入开会员赠礼福利玩法——用户开通美柚会员后，可领取专属实物赠礼作为会员权益。此举既能增强会员权益感知，也能提升开会员的转化动力。

1.2 业务子背景

运营侧已完成赠礼选品，需要一套完整的赠礼领取页承载从"浏览商品 → 确认地址 → 领取成功 → 物流追踪"的用户全链路。同时，后端需支持运营配置赠礼图片、商品信息、使用须知等内容。

1.3 现状判断及问题

现状问题判断历史需求解决方案
会员页仅有权益列表展示，无实物领取能力会员权益停留在虚拟权益（优惠券、积分），缺乏高感知的实物福利无新建赠礼详情页，承载实物领取全流程
无收货地址管理能力用户在美柚 App 内没有维护收货地址的路径无新建地址选择页，支持地址列表展示与选择
无赠礼物流追踪能力用户领取后无法查看发货状态和物流信息无页面内嵌物流模块，支持已发货/未发货双态切换
会员开通后无实物福利感知现有会员权益以虚拟权益为主，缺乏实物赠礼的高感知触点无赠礼卡片区域展示赠礼视觉素材，支持后台配置

二、项目目标

2.1 目标描述

  1. 运营可配置：运营可通过后台配置赠礼图片、商品标题、使用须知文案、发货时效，前端无需发版
  2. 用户体验闭环：用户可在赠礼详情页完成"浏览→领取→查物流"的一站式操作，减少跳转流失
  3. 降低人工客服成本：通过使用须知、物流自助查询、地址不可修改提示等功能，减少客服改址/催发货工单

2.2 迭代节奏

阶段内容说明
Phase 1赠礼详情页 MVP赠礼展示 + 商品标题 + 免费领取按钮 + 地址选择 + 确认流程 + 已领取状态 + 物流展示 + 使用须知
Phase 2后台配置能力运营后台配置赠礼图、商品信息、使用须知、发货天数
Phase 3物流自动同步对接快递公司 API，自动获取物流轨迹并在页内展示

2.3 风险预判

风险项级别说明应对策略
地址数据准确性高用户可能填错地址导致发错货二次确认弹窗 + 地址锁定后不可修改 + 客服电话兜底
库存不足中赠礼数量有限，领取超出库存"免费领取"按钮置灰 + "已领完"提示
非会员访问低非会员用户访问本页按钮置灰 + 引导开通会员
并发领取中高并发下超发后端做幂等校验，一人一单

3.6 需求说明

模块一：赠礼卡片

原型截图功能模块功能点优先级详细说明
赠礼卡片赠礼图片展示P0展示赠礼图片，由运营在赠礼后台配置并上传。图片原比例显示，带圆角和阴影效果，支持 jpg/png/webp 格式
赠礼卡片图片配置能力P1赠礼图片、图片替代文本（alt）由后台配置下发，前端不做文案编辑，仅渲染
赠礼卡片点击交互P2当前版本为纯展示；后续版本可扩展活动页跳转或详情查看

模块二：领取卡片

原型截图功能模块功能点优先级详细说明
领取卡片商品标题展示P0展示赠礼名称（如"美柚一心婴儿柔纸巾..."），标题支持多行，由后台配置
领取卡片免费领取按钮P0粉色胶囊按钮，居中展示。未领取用户可点击触发领取流程
领取卡片已领取状态P0确认领取后切换：按钮变灰显示"已领取"，下方展示预计寄出时间（如"预计7个工作日内寄出"）
领取卡片按钮防抖P1已领取状态不可逆；未领取状态点击后弹窗交互，防止重复点击
领取卡片异常状态-已领完P1库存不足时按钮置灰显示"已领完"，不可点击
领取卡片异常状态-非会员P2非会员用户按钮置灰 + 引导开通会员提示

模块三：收货地址展示

原型截图功能模块功能点优先级详细说明
收货地址常驻展示P0已领取后收货信息常驻展示在领取卡片下方，浅灰底色（#f6f6f8）区分于白色主卡片
收货地址信息展示规则P0收件人姓名完整展示；手机号中间4位脱敏（如 132****5555）；详细地址完整展示省市区+门牌号
收货地址只读锁定P0地址确认后锁定不可编辑，如有误需联系客服人工修改
收货地址数据来源P0数据来源于用户在地址选择页的选择，通过 URL 参数（name/phone/addr）传递回详情页

模块四：物流信息

原型截图功能模块功能点优先级详细说明
物流信息未发货状态P0显示"🚚 等待发货"，灰色文字，纯文本无操作
物流信息已发货状态P0显示快递公司名称 + 物流单号 + 粉色"复制"按钮

物流信息状态切换P1运营在后台录入快递公司和单号后，前端刷新即切换为已发货状态

原型截图功能模块功能点优先级详细说明

原型截图功能模块功能点优先级详细说明

模块七：二次确认弹窗

原型截图功能模块功能点优先级详细说明
二次确认弹出触发P0用户在确认地址浮层点击"确认领取"后弹出
二次确认弹窗内容P0居中模态框，⚠️图标 + "确认领取赠礼？"标题 + "领取后收货地址将不可修改"提示
二次确认取消按钮P0点击"取消"关闭二次弹窗，回到确认地址浮层

3.7 协同方需求

协同方配合内容备注
后端开发提供赠礼详情数据接口（赠礼图、商品标题、使用须知、发货天数）、用户收货地址列表接口、物流信息查询接口、领取/已领取状态查询接口需支持幂等，防止重复领取
运营配置赠礼图片素材、商品标题文案、使用须知内容、发货天数图片需提供规范尺寸和格式要求
客服准备地址修改、物流查询等话术模板需在后台系统开通订单查询权限
会员团队提供会员状态校验接口，确保非会员无法领取需与会员开通流程联动
