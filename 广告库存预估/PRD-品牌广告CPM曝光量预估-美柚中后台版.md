# 一、需求背景

## 1.1 业务大背景

品牌广告售卖场景中，运营人员需要在售前、排期前和方案评审阶段，快速判断某广告位在指定时间段内是否还有可售库存，以及已有交叉计划对库存和曝光的影响。当前业务需要一个可直接演示、可快速解释的预估工具，支撑广告售卖判断、库存沟通和内部评审。

该需求服务于广告商业化方向中“提升库存判断效率、降低沟通成本、增强方案评审效率”的目标。

## 1.2 业务子背景

当前“品牌广告CPM曝光量预估”页面已具备基础原型能力，但在使用和评审过程中暴露出以下问题：

- 字段名称不统一，存在“库存曝光转化比”与“库存曝光转化率”混用
- 结果区字段口径不清晰，CPM 与非 CPM 库存容易混淆
- 预估细节和预估结果的展示位置不符合业务理解顺序
- 说明文案不够直观，业务在评审时仍需额外解释计算关系
- 数值未统一千分位展示，影响可读性
- 表单默认值和必填规则与实际使用习惯不完全一致

当前阶段页面主要面向演示和评审原型，目标是先把结构、文案、流程和展示口径统一，再根据后续反馈决定是否产品化。

## 1.3 现状判断及问题

| 现状 | 问题判断 | 历史需求/反馈 | 解决方案 |
| --- | --- | --- | --- |
| 预估表单已存在基础字段 | 样式字段仍按必填理解，影响提交体验 | 用户提出“样式改为非必选” | 将样式字段改为非必填，并移除提交校验 |
| 手机平台支持多选 | 默认未体现“全选”的业务习惯 | 用户提出“手机平台，默认全选” | 默认勾选 IOS、Android、鸿蒙，并同步按钮文案 |
| 页面有预估结果展示 | 预估细节与预估结果展示顺序不清晰 | 用户要求“先展示预估细节，再展示预估结果” | 拆分两步展示流程，右侧先出细节，下方后出结果 |
| 右侧区域已存在信息展示 | 细节位置和高度不符合截图要求 | 用户多次调整位置与范围 | 将预估细节固定在页面右侧，并限制在标题到按钮区域内 |
| 结果表已有库存字段 | CPM 与非 CPM 口径混淆 | 用户持续要求增加非CPM字段、删除CPM说明 | 在计划表和分天表中新增非CPM字段，并统一说明文案 |
| 说明文案展示库存与 CPM | 不利于直接理解曝光计算关系 | 用户要求改为“剩余库存 * 转化率” | 将说明文案改成剩余库存乘库存曝光转化率 |
| 数值展示为普通数字 | 长数字可读性差 | 用户要求“数值按照千分位展示” | 所有核心结果字段按千分位展示 |
| 结果顶部信息独立显示 | 业务希望说明更贴近优先占量模块 | 用户要求移动到优先占量计划下方 | 将“初始库存/库存曝光转化率”移动到优先占量计划标题下方 |

# 二、项目目标

## 2.1 目标描述

本次迭代目标是把“品牌广告CPM曝光量预估”页面优化为一个可直接用于业务评审和库存沟通的中后台原型工具，重点提升：

1. **结果可理解性**：字段命名统一，CPM 与非 CPM 口径清晰，说明文案可直接表达计算关系
2. **页面可演示性**：预估细节与结果展示顺序符合业务理解，布局稳定，便于截图和讲解
3. **操作可用性**：表单默认值和校验逻辑更贴近业务使用习惯
4. **展示规范性**：所有关键数值按统一格式展示，页面整体输出更适合评审

## 2.2 迭代节奏

### 第一阶段：原型结构优化

- 统一文案和字段名
- 调整表单默认值和必填规则
- 重构预估细节与预估结果布局

### 第二阶段：结果口径优化

- 增加非CPM字段
- 调整说明文案
- 按千分位统一数值展示

### 第三阶段：评审版收口

- 补齐流程图、状态流转、验收用例
- 形成完整评审材料

## 2.3 风险预判

| 风险点 | 描述 | 影响 | 应对方案 |
| --- | --- | --- | --- |
| 口径理解偏差 | 业务对 CPM / 非 CPM 字段理解可能不同 | 影响评审一致性 | 在页面中直接显示公式和说明文案 |
| 原型与正式系统差异 | 当前页面为静态模拟，不接真实接口 | 影响后续产品化预期 | 在 PRD 中明确当前为原型演示版本 |
| 布局适配风险 | 当前布局主要针对桌面端 | 窄屏下可能展示异常 | 暂以桌面端评审为主，移动端后续确认 |

# 三、需求方案

## 3.1 名词定义

| 名词 | 定义 |
| --- | --- |
| 库存曝光转化率 | 库存转化为曝光的比例，用于计算曝光交叉占量和剩余曝光 |
| 库存交叉占量 | 被交叉计划占用的 CPM 口径库存 |
| 库存交叉占量（非CPM） | 库存交叉占量乘 1000 后得到的非 CPM 口径库存 |
| 剩余库存 | 扣减计划占量后剩余的 CPM 口径库存 |
| 剩余库存（非CPM） | 剩余库存乘 1000 后得到的非 CPM 口径库存 |
| 曝光交叉占量 | 库存交叉占量乘库存曝光转化率后的曝光值 |
| 剩余曝光 | 剩余库存乘库存曝光转化率后的曝光值 |
| 优先占量计划 | 优先级更高、优先扣减库存的交叉计划 |
| 预定计划 | 在优先占量计划之后再参与扣减的计划 |

### 3.6.2 表单区说明

#### 5.1 预估标题

- 类型：文本输入框
- 规则：必填，最多 32 字
- 页面展示：带字数统计

#### 5.2 广告位信息

包含：

- 应用（必填）
- 页面（必填）
- 广告位（必填）
- 样式（非必填）
- 子位（必填，范围 1~9）
- 手机平台（必填，多选，默认全选）
- 售卖方式（必填）
- 投放起止时间（必填）
- 最小版本（非必填）
- 最大版本（非必填）
- 库存曝光转化率（必填）

校验规则：

- 预估标题为空：不可提交
- 投放起止时间为空：不可提交
- 库存曝光转化率为空：不可提交
- 手机平台未选择：不可提交
- 样式为空：允许提交

校验文案：

> 请补全预估标题、库存曝光转化率、手机平台和投放起止时间后再进行预估。

### 3.6.3 预估细节区说明

- 触发时机：点击“预估曝光”且校验通过后立即展示
- 展示位置：页面右侧
- 展示内容：
  - 标题“预估细节”
  - 主位置说明
  - 交叉计划表格
- 展示高度：限制在“预估标题”到“预估曝光按钮”范围内
- 内容溢出：区域内部滚动

表格字段：

- 交叉计划ID
- 计划信息
- 投放细节
- 状态
- 说明

### 3.6.4 预估结果区说明

#### 结果总览

展示内容：

- 预估结果摘要
- 剩余曝光总量
- 初始库存

说明：

- 独立“预估结果”标题去除
- “初始库存 / 库存曝光转化率”整段移动至优先占量计划标题下方

#### 优先占量计划

表头字段：

1. 交叉计划Id
2. 广告主
3. 计划投放类型
4. 计划来源
5. 库存交叉占量（非CPM）
6. 库存交叉占量
7. 曝光交叉占量
8. 投放时间

规则：

- 计划 ID 按数值升序展示
- 广告主展示示例为“康臣药业”
- 标题带注释 icon
- 注释文案：`计划id数值越小优先级越高，越先占量`

说明文案格式：

- `剩余曝光：xx,xxx （剩余库存：xx,xxx * 0.8928）`

#### 预定计划

表头字段：

1. 交叉计划Id
2. 广告主
3. 计划投放类型
4. 库存交叉占量（非CPM）
5. 库存交叉占量
6. 曝光交叉占量
7. 投放时间

规则与优先占量计划一致。

#### 分天剩余结果

字段：

1. 日期
2. 剩余库存（非CPM）
3. 剩余库存
4. 剩余曝光（剩余库存 * 库存曝光转化率）

规则：

- `剩余库存（非CPM） = 剩余库存 * 1000`
- 去掉“合计”行
- 数值统一千分位展示

### 3.6.5 计算规则

| 规则项 | 公式 |
| --- | --- |
| 库存交叉占量（非CPM） | 库存交叉占量 * 1000 |
| 剩余库存（非CPM） | 剩余库存 * 1000 |
| 曝光交叉占量 | 库存交叉占量 * 库存曝光转化率 |
| 剩余曝光 | 剩余库存 * 库存曝光转化率 |

### 3.6.6 验收标准

| 验收项 | 验收内容 |
| --- | --- |
| 表单规则 | 样式非必填；手机平台默认全选 |
| 流程顺序 | 点击预估后先出细节，后出结果 |
| 布局 | 细节在右侧，结果在下方全宽 |
| 结果表 | 增加广告主列、非CPM字段、去掉合计 |
| 说明文案 | 改为“剩余库存 * 库存曝光转化率” |
| 数值格式 | 按千分位展示 |

# 四、附录

## 4.1 当前范围说明

当前版本定位为原型评审版，仅覆盖单页预估能力，不包含：

- 真实接口对接
- 结果保存
- 数据导出
- 审批流程
- 权限控制

## 4.2 后续建议

后续若进入产品化阶段，建议继续补充：

- 接口设计
- 字段字典
- 异常态细化
- 角色权限模型
- 与广告位管理、计划管理的上下游联动
