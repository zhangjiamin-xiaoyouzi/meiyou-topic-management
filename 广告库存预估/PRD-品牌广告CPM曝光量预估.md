# PRD：品牌广告CPM曝光量预估

## 1. Summary

本文档用于定义“品牌广告CPM曝光量预估”页面的产品需求。该页面面向广告投放运营人员，帮助其在创建品牌广告投放方案前，快速完成库存、交叉占量、剩余库存和剩余曝光的预估，并查看预估细节与结果说明。

本次 PRD 基于现有页面功能与本轮聊天记录中的需求迭代整理，重点覆盖表单输入、预估细节展示、预估结果展示、字段口径、交互流程与展示规则。

## 2. Contacts

| 姓名 | 角色 | 备注 |
| --- | --- | --- |
| 需求方 | 产品/业务 | 提出字段、文案、交互和展示口径调整需求 |
| 设计方 | UI/交互设计 | 负责“预估细节”区域样式与页面布局规范 |
| 开发方 | 前端开发 | 负责静态页面实现、交互逻辑与结果渲染 |
| 使用方 | 广告投放运营 | 使用页面完成库存曝光预估与计划占量分析 |

## 3. Background

### 3.1 项目背景

品牌广告投放前，运营人员需要快速判断某广告位在指定时间范围内是否还有可售库存，以及不同类型交叉计划对库存的占用情况。原有页面需要逐步补齐结果信息展示、字段口径说明和交互流程，才能支撑业务演示和方案评估。

### 3.2 为什么现在要做

当前页面已作为业务预估工具原型使用，但在实际演示和评审过程中暴露出以下问题：

- 字段名称不统一，如“库存曝光转化比”需要统一为“库存曝光转化率”
- 结果展示口径不够清楚，CPM 与非 CPM 的库存字段容易混淆
- 预估细节和预估结果的展示区域、顺序和大小不符合使用预期
- 表单默认值和必填规则需要更贴合业务操作习惯
- 数值展示格式不统一，不便于阅读和汇报

### 3.3 变化点

本轮迭代后，页面已经支持：

- 两步式预估展示流程
- 页面右侧展示“预估细节”
- 页面下方全宽展示“预估结果”
- 交叉计划表、分天剩余表、说明文案的口径统一
- 手机平台默认全选
- 样式改为非必填
- 数值按千分位展示

## 4. Objective

### 4.1 目标

提供一个清晰、直观、可演示的品牌广告 CPM 曝光量预估页面，让运营人员在填写广告位与投放条件后，可以快速看到：

- 交叉计划明细
- 优先占量计划与预定计划的占量情况
- 分天剩余库存和剩余曝光
- 关键计算口径说明

### 4.2 业务价值

对公司：

- 提升广告库存评估效率
- 降低人工口径解释成本
- 支撑销售、运营、产品内部沟通和方案评审

对用户：

- 更快判断广告位是否可售
- 更容易理解“库存”“曝光”“交叉占量”的关系
- 更方便对外汇报和截图展示

### 4.3 与策略的关系

该页面属于广告投放运营后台中的品牌小工具能力，服务于品牌广告售卖、库存预估和投放评估场景，是中后台运营提效的重要工具。

### 4.4 Key Results

#### KR1：提升预估效率
- 在必填信息完整的前提下，用户可在一次点击后完成预估流程
- 用户从填写到看到完整结果的操作路径不超过 2 步

#### KR2：提升结果可理解性
- 结果区所有核心数值统一为千分位展示
- CPM 与非 CPM 口径字段均有明确命名
- 关键说明文案可直接解释计算关系

#### KR3：提升页面可演示性
- 预估细节区域位置、大小和结构清晰稳定
- 预估结果区域横向铺满，不出现明显留白和压缩问题
- 页面支持直接用于评审演示与截图输出

## 5. Market Segment(s)

### 5.1 目标用户

本功能面向以下用户：

- 广告投放运营
- 品牌广告产品经理
- 广告销售支持人员
- 需要评估广告位剩余可售库存的业务人员

### 5.2 用户要解决的问题

用户不是要“看一个页面”，而是要解决以下工作问题：

- 某广告位在某时间段还有多少可售库存
- 当前库存被哪些交叉计划占用
- 优先占量计划和预定计划分别影响多少库存
- 剩余库存能转化成多少曝光
- 是否适合继续售卖或排期

### 5.3 使用约束

- 页面为中后台工具，优先满足业务清晰度，不追求复杂自动化
- 当前数据以页面内置示例和模拟逻辑为主
- 结果用于辅助判断，不作为最终结算依据

## 6. Value Proposition(s)

### 6.1 用户价值

该功能帮助用户：

- 快速完成预估，而不是人工拆表计算
- 一眼看到交叉计划影响，而不是手动逐个排查
- 直接理解“库存 → 曝光”的换算关系，而不是口头解释
- 在演示中展示规范结果，而不是临时整理截图和文案

### 6.2 缓解的痛点

- 痛点 1：字段名和展示口径混乱
  - 解决方式：统一“库存曝光转化率”命名，区分 CPM / 非 CPM

- 痛点 2：预估流程割裂
  - 解决方式：先展示预估细节，再展示预估结果

- 痛点 3：结果不易阅读
  - 解决方式：所有关键数值千分位展示，结构分块清晰

- 痛点 4：业务解释成本高
  - 解决方式：通过注释 icon 和说明文案直接表达规则与公式

### 6.3 相对替代方案的优势

相较人工表格或口头说明，本工具具备：

- 更统一的字段口径
- 更直观的页面布局
- 更低的解释成本
- 更适合业务评审和演示

## 7. Solution

### 7.1 UX / Prototype

#### 页面结构

页面分为三部分：

1. 左侧表单区
   - 用于填写预估标题、广告位信息、定向信息、投放限制等
2. 右侧预估细节区
   - 点击“预估曝光”后先展示
   - 用表格展示交叉计划明细
3. 下方预估结果区
   - 等待预估完成后展示
   - 全宽显示，不被左右分栏压缩

#### 交互流程

1. 用户填写基础信息
2. 用户点击“预估曝光”
3. 页面右侧先展示“预估细节”
4. 页面显示预估中状态
5. 预估完成后，页面下方展示“预估结果”

### 7.2 Key Features

#### 功能 1：表单输入与校验

支持以下输入项：

- 预估标题（必填）
- 应用（必填）
- 页面（必填）
- 广告位（必填）
- 样式（非必填）
- 子位（必填，步进器，范围 1~9）
- 手机平台（必填，默认全选 IOS / Android / 鸿蒙）
- 售卖方式（必填）
- 投放起止时间（必填）
- 最小版本（非必填）
- 最大版本（非必填）
- 库存曝光转化率（必填）
- 定向信息（身份、年龄、地区、人群开关）
- 投放限制（限制开关、时间类型、单用户限制）

校验规则：

- 若预估标题、投放起止时间、库存曝光转化率、手机平台未填写完整，则不允许提交
- 样式不参与必填校验

校验提示文案：

- “请补全预估标题、库存曝光转化率、手机平台和投放起止时间后再进行预估。”

#### 功能 2：预估细节展示

点击“预估曝光”后，页面右侧展示“预估细节”。

展示要求：

- 位于页面右侧
- 高度范围限制在“预估标题”到“预估曝光按钮”之间
- 超出内容可滚动
- 标题为“预估细节”
- 展示主位置
- 展示表格字段：
  - 交叉计划ID
  - 计划信息
  - 投放细节
  - 状态
  - 说明

状态说明示例：

- 交叉占量
- 不占量（预定计划不占量）
- 不交叉（预估开始前完成）

#### 功能 3：预估结果总览

预估完成后，在页面下方展示结果总览。

展示内容：

- 预估结果摘要
- 剩余曝光总量
- 初始库存
- 两类计划块：
  - 优先占量计划
  - 预定计划

说明：

- “预估结果”单独标题已去掉
- “初始库存 / 库存曝光转化率”文案移动到“优先占量计划”标题下方

#### 功能 4：优先占量计划表

字段包括：

- 交叉计划Id
- 广告主
- 计划投放类型
- 计划来源
- 库存交叉占量（非CPM）
- 库存交叉占量
- 曝光交叉占量
- 投放时间

规则：

- 计划 ID 按数值升序排列
- “库存交叉占量（非CPM） = 库存交叉占量 * 1000”
- “曝光交叉占量 = 库存交叉占量 * 库存曝光转化率”
- 广告主展示示例为“康臣药业”

补充说明：

- 标题后提供注释 icon
- 注释文案：`计划id数值越小优先级越高，越先占量`

#### 功能 5：预定计划表

字段包括：

- 交叉计划Id
- 广告主
- 计划投放类型
- 库存交叉占量（非CPM）
- 库存交叉占量
- 曝光交叉占量
- 投放时间

规则与优先占量计划一致：

- 计划 ID 按数值升序排列
- 非 CPM 字段按 `*1000` 计算
- 曝光交叉占量按转化率计算

补充说明：

- 标题后提供注释 icon
- 注释文案同优先占量计划

#### 功能 6：剩余结果说明文案

两类计划下方均展示剩余结果说明。

展示格式：

- `剩余曝光：93,915 （剩余库存：105,191 * 0.8928）`

要求：

- 不再展示“剩余库存(CPM)”文案
- 直接展示“剩余库存 * 库存曝光转化率”的计算关系

#### 功能 7：分天剩余库存与曝光表

优先占量计划、预定计划下方均展示分天结果表。

字段包括：

- 日期
- 剩余库存（非CPM）
- 剩余库存
- 剩余曝光（剩余库存 * 库存曝光转化率）

规则：

- `剩余库存（非CPM） = 剩余库存 * 1000`
- `剩余曝光 = 剩余库存 * 库存曝光转化率`
- 去掉“合计”一行
- 数值统一按千分位展示

#### 功能 8：字段与文案统一

文案统一要求：

- “库存曝光转化比”统一改为“库存曝光转化率”
- 样式字段改为非必填
- 手机平台默认全选
- 数值统一按千分位展示

### 7.3 Technology

当前为静态 HTML 原型页面，技术方案如下：

- 单文件实现：HTML + CSS + 原生 JavaScript
- 主要通过 DOM 渲染表格和说明文案
- 通过 `setTimeout` 模拟预估等待过程
- 通过内置示例数据生成交叉计划和剩余结果

当前不包含：

- 后端接口调用
- 实时库存计算服务
- 权限和日志上报能力

### 7.4 Assumptions

当前需求基于以下假设：

1. 当前页面主要用于原型演示和需求确认，不是正式生产系统
2. 交叉计划数据和剩余库存计算逻辑可由静态示例模拟
3. 运营人员更关心结果清晰度，而不是复杂高级筛选
4. 结果的业务解释需要直接显示在页面中，而不是依赖培训
5. 非 CPM 口径字段用于帮助业务理解真实库存规模

## 8. Release

### 8.1 建议拆分

#### V1：基础可用版本

包含：

- 表单输入与基础校验
- 右侧预估细节展示
- 下方预估结果展示
- 优先占量计划 / 预定计划表
- 分天剩余库存与曝光表
- 文案统一与千分位格式化
- 平台默认全选
- 样式非必填

#### V2：增强版本

可继续补充：

- 接入真实库存与计划数据接口
- 增加更多筛选条件
- 支持导出结果
- 支持真实 loading 状态与失败重试
- 支持不同广告位模板和字段差异

### 8.2 工作量预估

若继续在现有静态页面基础上迭代：

- V1 调整：短周期完成
- V2 接口化和产品化：中周期完成

### 8.3 上线建议

建议先以原型评审版本继续收集业务反馈，再决定是否：

- 接入真实数据源
- 纳入正式中后台模块
- 增加导出、日志、权限等企业级能力
